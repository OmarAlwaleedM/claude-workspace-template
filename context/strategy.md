# Strategy

## Current Focus Period

March 2026 — Python course final project (presentation date TBD)

## Strategic Priorities

1. **Build "Economy Collapse Speedrun"** — a multiplayer, Kahoot-style Python game where the class collectively tries to destroy a fictional economy as fast as possible
2. **Keep it technically simple** — Python-first, minimal HTML/frontend, lean on libraries
3. **Make it engaging and funny** — edgy policy choices generated by an LLM, calibrated to make 20-year-olds laugh without getting Omar in trouble with the professor
4. **Nail the 5-minute demo** — QR code on the big screen, everyone joins on their phones, chaos ensues

## What Success Looks Like

- The game runs smoothly during the live demo with the whole class participating
- People are laughing and engaged picking absurd policies
- The professor sees a legit Python project with real architecture (web server, API calls, game logic, scoring)
- Omar gets a good grade and maybe a round of applause

## Key Decisions & Open Questions

- Exact Python framework for the web/multiplayer piece (Flask? FastAPI + simple frontend?)
- How to generate + display the QR code on the projector screen
- OpenRouter model choice and system prompt tuning for the right level of edginess
- How many rounds / how scoring and pacing work within a configurable time limit
- How to keep the phone UI dead simple (just vote on 1 of 4 options)
